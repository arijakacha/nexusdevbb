🔐 Password Reset Test Guide

This guide will help you test the password reset functionality step by step.

📋 Step 1: Configure Email

⚠️ Action Required: Update your .env file with real Gmail credentials

Current configuration in .env:

MAILER_DSN=gmail://your-email@gmail.com:your-app-password@default

To setup Gmail:

  1. Enable 2-factor authentication on your Gmail account
  2. Go to Google Account settings → Security → App passwords
  3. Generate a new app password
  4. Replace in .env: your-email@gmail.com:your-app-password

🧪 Step 2: Test Password Reset

Step 2.1: Go to the forgot password page
http://localhost:8000/forgot-password
Step 2.2: Enter an email address of an existing user
  • Use a real user email from your database
  • Click "Send reset link"
Step 2.3: Check the email
  • You should receive an email at the entered address
  • Subject: "Reset your NexusPlay password"
  • Contains a reset button and link
Step 2.4: Click the reset link
  • Opens the reset password page
  • Link expires in 30 minutes
Step 2.5: Enter new password
  • Choose a new password
  • Confirm the password
  • Click "Reset password"

✅ What Should Work

  • Forgot password form loads properly
  • Email is sent with reset link
  • Reset link is valid for 30 minutes
  • Reset form loads with valid token
  • Password is successfully updated
  • User can login with new password
  • Invalid/expired tokens show error message

🛠️ Troubleshooting

If email doesn't send:
  • Check Gmail credentials in .env
  • Ensure 2FA is enabled on Gmail
  • Use an App password (not regular password)
If reset link doesn't work:
  • Check if link expired (30 minutes)
  • Ensure token wasn't already used
  • Check URL is copied correctly
Debug in Symfony:
php bin/console debug:mailer

🔍 Development Testing

If you don't want to send real emails during development, you can use:

MAILER_DSN=smtp://127.0.0.1:1025

Then install MailHog to catch emails:

docker run -p 1025:1025 -p 8025:8025 mailhog/mailhog

Visit http://localhost:8025 to see emails
