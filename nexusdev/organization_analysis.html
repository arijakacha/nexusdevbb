🏢 Organization System Analysis

Complete analysis of the NexusPlay Organization Management System

📋 Entity Structure

Organization Entity

Fields: - id (int, auto-generated) - name (string, 150 chars, required) - description (text, optional) - logo (string, 255 chars, optional) - isValidated (boolean, default: false) - createdAt (DateTimeImmutable, auto-set) - deletedAt (DateTimeImmutable, nullable) - owner (User, required, many-to-one) - teams (Team collection, one-to-many)

Relationships

  • Owner: One User owns multiple Organizations
  • Teams: One Organization has multiple Teams
  • Soft Delete: Uses deletedAt timestamp
  • Validation: Boolean isValidated flag

🎮 Controllers

OrganizationController (Back Office)

Route: /BOrganization
  • Main Route: app_organization_back
  • Views: dashboard, profile, teams, players, analytics, leaderboards, shop, notifications
  • Admin Mode: ROLE_ADMIN can manage all organizations
  • User Mode: Regular users manage their own organization

FrontOrganizationController (Public)

Route: /FOrganization
  • Public Listing: Browse all organizations
  • Search & Filter: By name, verification status
  • Sorting: By name, creation date
  • Pagination: 12 items per page

⚠️ Issues & Improvements Needed

🔧 Database Schema Issues

[FAIL] The entity-class App\Entity\Organization mapping is invalid: * The field App\Entity\Organization#teams is on the inverse side of a bi-directional relationship, but the specified mappedBy association on the target-entity App\Entity\Team#organization does not contain the required 'inversedBy: "teams"' attribute.

Solution: Add inversedBy="teams" to Team entity's organization mapping

🗄️ Database Sync Required

The database schema is not in sync with the current mapping files.
